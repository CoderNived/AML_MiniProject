Build & execution steps


  
1
Place the 74HC595. Insert it straddling the center gap of the breadboard. Connect the power rails: 3.3 V from LaunchPad J1-1 and GND from J1-20 to the breadboard bus strips.


  
2
Wire the shift register control pins. Connect SRCLR (pin 10) to 3.3 V and OE (pin 13) to GND. Run jumper wires from LaunchPad P1.1 → SER, P1.2 → SRCLK, P1.3 → RCLK.


  
3
Wire the LED array. Place 8 LEDs in a row. Connect each anode through a 220 Ω resistor to 74HC595 outputs QA (pin 15) through QH (pin 7). Connect all cathodes to GND.


  
4
Connect the MAX9814. VDD → 3.3 V, GND → GND, OUT → P1.0. For maximum sensitivity connect the GAIN pin to GND (60 dB). Leave AR floating.


  
5
Install CCS or Energia. In CCS, go to File → New → CCS Project, set Target = MSP430G2553, select Empty Project (main.c). In Energia, go to Tools → Board → MSP-EXP430G2553LP.


  
6
Paste and build the code. Copy the code from the Code tab. Replace main.c with it. Press Ctrl+B in CCS to build. Errors here usually mean a wrong target device selected.


  
7
Flash to the LaunchPad. Connect via USB. In CCS press F5 (Debug) — this flashes the firmware and halts at main(). Press F8 (Resume) to run. In Energia just click Upload (→).


  
8
Test. Speak, clap, or play music near the microphone. LEDs should light up progressively from LED1 (quiet) to LED8 (loud). If nothing lights up, confirm OUT → P1.0 and that GAIN is connected to GND.


  
9
Calibrate thresholds. Edit the levelToLED() function in the code. Lower the threshold numbers to make the meter more responsive; raise them if LEDs are always fully lit. The thresholds map ADC amplitude deviation (0 – 512) to 8 bars.


  
Energia users: The code uses raw MSP430 register names (ADC10CTL0, WDTCTL, etc.) that Energia passes through directly — no modifications needed. Just compile and upload.

  
Voltage warning: Keep everything at 3.3 V. The LaunchPad's GPIO and ADC pins are rated to VCC (3.3 V). A 5 V signal on P1.0 or P1.1 will damage the device permanently.
